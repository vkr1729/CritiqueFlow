### 🐛 Bugfix Plan

#### Bug 1: `_read_text` returns empty string when `FILE_CHAR_CAP=0` (unlimited)
- **File:** `audit-harness/core/file_reader.py`
- **Line(s):** 34
- **Root Cause:** The new default `FILE_CHAR_CAP=0` means "unlimited", but `f.read(0)` in Python returns an empty string — the guard `char_cap >= sys.maxsize` is `False` when `char_cap` is `0`, so it falls through to `f.read(0)`.
- **Fix:** Change the condition to also treat `0` (and negative values like `-1`) as "read all":
  ```python
  content = f.read() if char_cap <= 0 or char_cap >= sys.maxsize else f.read(char_cap)
  ```

#### Bug 2: `renderSummaryDashboard` receives wrong data shape from `loadSession`
- **File:** `audit-harness/web/static/app.js`
- **Line(s):** ~541–542 (inside the `loadSession` function, the calls to `renderSummaryDashboard`)
- **Root Cause:** `loadSession` calls `renderSummaryDashboard(data.chain)`, passing the chain object directly. But `renderSummaryDashboard` internally does `var chain = data.chain`, expecting the *wrapping* API response — so it ends up with `chain = undefined`, `steps = []`, and no confidence/metrics are displayed for restored sessions.
- **Fix:** In `loadSession`, wrap the chain so the data shape matches what `renderSummaryDashboard` expects:
  ```js
  renderSummaryDashboard({
    chain: data.chain,
    total_iterations: data.chain.total_iterations,
    early_stopped: data.chain.early_stopped,
    referenced_files: data.chain.referenced_files
  });
  ```
  Alternatively, change the two call sites to be consistent (adjust `renderSummaryDashboard` to accept a chain directly and update the `runQuery` call site accordingly).
